                                                                    # Map 和 Set

                                                                    1. Map 相当于普通对象更为灵活，普通对象的 key 只可以以基础数据类型作为 key 值，并且所有传入的 key 值都会转为 string，而 Map 的 key 可以是各种数据类型
                                                                    2. Set 是一个不重复的集合

                                                                    # 原型链

                                                                     1、当一个对象查找属性和方法时会从自身查找,如果查找不到则会通过__proto__指向被实例化的构造函数的prototype

 2、隐式原型也是一个对象,是指向我们构造函数的原型

 3、除了最顶层的Object对象没有__proto_，其他所有的对象都有__proto__,这是隐式原型

 4、隐式原型__proto__的作用是让对象通过它来一直往上查找属性或方法，直到找到最顶层的Object的__proto__属性，它的值是null,这个查找的过程就是原型链

 5、实例化对象的constructor指向构造函数的原型的constructor，即 person.constructor = Person.prototype.constructor

                                                                    # 箭头函数和普通函数有什么区别?

                                                                    1. 箭头函数更加简洁

                                                                    2. 箭头函数没有自己的this

                                                                      它不会创建自己的this，只会在上一层作用域继承this,在指定时就定义了，不可以改变

                                                                      普通函数谁调用就是指向谁,可以改变 this

                                                                    3. 箭头函数不可以作为构造函数使用，即不可以 new

                                                                    4. 箭头函数没有自己的 arguments

                                                                    5. 箭头函数没有 prototype

                                                                    6. 箭头函数不能用作 Generator 函数，不可以使用 yeild 关键字

                                                                    # 什么是闭包，闭包的作用是什么，以及闭包使用场景

                                                                    当一个内部函数被调用，就会形成闭包，闭包是指能够读取其它函数内部的变量

                                                                    闭包作用：

                                                                    局部变量可以长久保存又不会被污染

                                                                    封装对象的私有属性和私有方法

                                                                    闭包缺陷： 容易造成内存溢出

                                                                    闭包的使用场景：柯里化函数，使用闭包模拟私有方法,单例模式

                                                                    # 柯里化

                                                                    柯里化（currying），是把接受多个参数的函数变换成接受一个单一参数（最初函数的第一个参数）的函数，并且返回接受余下的参数而且返回结果的新函数的技术。

                                                                    # 单例模式

                                                                    function Singleton() {
  this.data = 'singleton'
}
Singleton.getInstance = function () {
  var instance
  return function () {
    if (instance) {
      return instance
    } else {
      instance = new Singleton()
      return instance
    }
  }
}
var a = Singleton.getInstance()
var b = Singleton.getInstance()
console.log(a === b) // true
console.log(a.data) // 'singleton'

                                                                    # New 操作符做了什么事情?

                                                                    1. 创建一个新对象
                                                                    2. 设置原型，将对象的原型设置为函数的 prototype 对象
                                                                    3. 让函数的this指向这个对象，执行构造函数的代码（为这个对象添加属性
                                                                    4. 判断函数的返回值类型，如果是值类型，返回创建的对象，如果是引用类型，就返回引用类型的对象

                                                                    # 说一下事件循环 Event Loop

                                                                    js 代码主要分为两大类： 同步代码、异步代码

                                                                    异步代码又分为：微任务与宏任务

                                                                    1.进入到script标签,就进入到了第一次事件循环.

2.遇到同步代码，立即执行

3.遇到宏任务,放入到宏任务队列里.

4.遇到微任务,放入到微任务队列里.

5.执行完所有同步代码

6.执行微任务代码

7.微任务代码执行完毕，本次队列清空

寻找下一个宏任务，重复步骤1

以此反复直到清空所有宏任务，这种不断重复的执行机制，就叫做事件循环
                                                                    1. 宏任务（macro task）：
                                                                      • 整个 Script 代码
                                                                      • 定时器 setTimeOut、SetImmediate
                                                                      • setImmediate (Node 独有)
                                                                      • ajax
                                                                      • 回调函数
                                                                      • Node 中 fs 可以进行异步的 I/O 操作
                                                                    2. 微任务（micro task）：
                                                                      • promise.nextTick,
                                                                      • promise.then()
                                                                      • async await
                                                                    3. 直接执行：
                                                                      • console.log()
                                                                      • new Promise 里面的

                                                                    易错点 (1). promise 本身是一个同步的代码(只是容器)，只有它后面调用的 then()方法里面的回调才是微任务

                                                                    (2). await 右边的表达式还是会立即执行,表达式之后的代码才是微任务, await 微任务可以转换成等价的 promise 微任务分析

                                                                    (3). script 标签本身是一个宏任务， 当页面出现多个 script 标签的时候，浏览器会把 script 标签作为宏任务来解析

                                                                    # script 标签中， async 和 defer 两个属性有什么用途和区别？

                                                                    在 html 中，有三种 script 标签

                                                                    <script src="xxx"></script>
<script src="xxx" async></script>
<script src="xxx" defer></script>

                                                                    script 标签用于加载脚本和执行脚本，直接使用 script 脚本时，

                                                                    html 会按照顺序加载执行脚本，在加载或者执行脚本过程中，会阻塞后续 DOM 渲染

                                                                    script 标签提供了两种方式来解决问题，就是加入属性 async 以及 defer，这两个属性使得 script 标签加载都不会阻塞 DOM 的渲染。

                                                                    defer：浏览器会异步下载该文件，不会影响后续渲染

		如果有多个defer的标签存在，会顺序执行所有的script，defer脚本会在文档渲染完毕之后，DOMContentLoaded事件调用前执行


async：使得script标签异步加载的情况下执行，不会按照script标签在页面上的顺序执行，而是谁加载完谁立即执行。

                                                                      function deepClone(obj, hash = new WeakMap()) {
  if (obj === null) return obj // obj如果为null或undefined ,则返回
  if (obj instanceof Date) return new Date(obj)
  if (obj instanceof RegExp) return new RegExp(obj)
  // 可能是对象或者普通的值  如果是函数的话是不需要深拷贝
  if (typeof obj !== 'object') return obj
  // 是对象的话就要进行深拷贝
  if (hash.get(obj)) return hash.get(obj)
  let cloneObj = new obj.constructor()
  // 找到的是所属类原型上的constructor,而原型上的 constructor指向的是当前类本身
  hash.set(obj, cloneObj)
  for (let key in obj) {
    if (obj.hasOwnProperty(key)) {
      // 实现一个递归拷贝
      cloneObj[key] = deepClone(obj[key], hash)
    }
  }
  return cloneObj
}
let obj = {name: 1, address: {x: 100}}
obj.o = obj // 对象存在循环引用的情况
let d = deepClone(obj)
obj.address.x = 200
console.log(d)

                                                                    function deepClone(source){
	const targetObj = source.constructor === Array? [] : {}
	for(const keys in source){
		if(source[keys].hasOwnPerporty){
			if(source[keys] && typeof source[keys] ==='object'){
				targetObj[keys] = source[keys].constructor === Array? []: {}
				targetObj[keys] = deepClone(source[keys])
			}else{
				targetObj[keys] = source[keys]
			}
		}
	}
	return targetObj[keys]
}

                                                                      • 3.如图片不能看见，window.innerHight > getBoundingClientRect().top
                                                                      • 4.如图片可以看见，window.innerHight < getBoundingClientRect().top
                                                                      // 获取所有图片
const images = document.querySelectorAll('img')
// 添加鼠标滚动事件
window.addEventListener('scroll', e => {
  images.forEach(image => {
    // 获取图片到视窗上的距离
    const imageTop = image.getBoundingClientRect().top
    // 如果图片可以看见
    if (imageTop < window.innerHeight) {
      // 把img标签中的data-src属性的值获取
      const data_src = image.getAttribute('data-src')
      // 添加 src属性
      image.setAttribute('src', data_src)
    }
  })
})

                                                                    2. IntersectionObserve

                                                                      • 创建一个观察实例const observer= new IntersectionObserve()
                                                                      • IntersectionObserve 接受两个参数,IntersectionObserve(callback),每次进入或者离开都会触发该 callback
                                                                      • 观察哪个 DOM 节点 observer.observe(dom节点)
                                                                      • 取消观察 observer.unobserve(dom节点)
                                                                    const images = document.querySelectorAll('img')
const callback = entries => {
  entries.forEach(entry => {
    if (entry.target) {
      const image = entry.target
      const data_src = image.getAttribute('data-src')
      image.setAttribute('src', data_src)
      //取消观察
      observer.unobserve(image)
    }
  })
}
//创建一个观察实例
const observer = new IntersectionObserver(callback)
// 观察哪个DOM节点
images.forEach(image => {
  observer.observe(image)
})

<script>
  const button = document.querySelector('button')
  function callback() {
    console.log('111')
  }
  function debounce(func, delay) {
    let timer
    return function () {
      // this指向callback函数
      let that = this
      // arguments 指向callback函数
      let args = arguments
      // 清除定时器
      clearTimeOut(timer)
      timer = setTimeout(() => {
        func.apply(that, args)
      }, delay)
    }
  }

  // 添加点击事件，传入防抖函数
  button.addEventListener('click', debounce(callback, 1000))
</script>

/** 节流函数 */
function throttle(func, delay) {
  let timer
  return function () {
    /** 如果timer存在，直接return */
    if (timer) {
      return
    }
    /** this指向和参数问题 */
    let that = this
    let args = arguments
    timer = setTimeOut(() => {
      func.apply(that, args)
      timer = null
    }, delay)
  }
}

window.addEventListener('resize', throttle(coloring, 1000))

                                                                    function coloring() {
  let r = Math.floor(Math.random() * 255)
  let g = Math.floor(Math.random() * 255)
  let b = Math.floor(Math.random() * 255)
  document.body.style.background = `rgb(${r},${g},${b})`
  console.log(`rgb(${r},${g},${b})`)
}
function throttle(func, delay) {
  let pre = 0
  return function () {
    /** 现在的时间戳 */
    let now = new Date()
    /** this指向和参数问题 */
    let that = this
    let args = arguments
    /** 现在的时间戳 - 上一次的时间戳  > 延迟毫秒数  则执行函数 */
    if (now - pre > delay) {
      func.apply(that, args)
      // 上一次的时间戳 覆盖为本次的时间
      pre = now
    }
  }
}
window.addEventListener('resize', throttle(coloring, 2000))

                                                                    # 冒泡排序

                                                                    const arr1 = [1, 51, 31, 561, 231, 5100, 2, 3, 1456]
function bubbleSort(arr) {
  let temp
  for (let i = 0; i < arr.length - 1; i++) {
    for (let j = 0; j < arr.length - 1 - i; j++) {
      if (arr[j] > arr[j + 1]) {
        temp = arr[j]
        arr[j] = arr[j + 1]
        arr[j + 1] = temp
      }
    }
  }
  return arr
}
console.log(bubbleSort(arr1)) // [1, 2, 3, 31, 51, 231, 561, 1456, 5100]
